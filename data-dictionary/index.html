    Data Dictionary

    This reference documents every table and column in the Cubby customer-facing analytics data warehouse. The warehouse is hosted in Google BigQuery and is refreshed daily.

    All tables live in the analytics dataset. Access is granted per organization — you will only see data for facilities belonging to your organization.

    Each table section below lists its columns, data types, and descriptions. Tables marked snapshot contain one row per entity per day; all others contain one row per event or entity.

    analytics

    The analytics dataset is the primary customer-facing schema. It is rebuilt nightly from the operational database and is designed for reporting and self-serve analysis.

    access_log

    analytics Logs every access control event (gate/door entry attempts) at a facility. Column-level documentation coming soon.

    auctions

    analytics One row per auction event held at a facility (typically lien auctions for delinquent tenants). Column-level documentation coming soon.

    book_entries

    analytics One row per accounting ledger entry. Includes charges, payments, credits, and transfers. The pre-calculated amt_* columns make it easier to aggregate common financial metrics. Column-level documentation coming soon.

    call_agent_availability

    analytics One row per manager per availability status per day. Tracks how long agents spent in each phone availability state. Column-level documentation coming soon.

    call_child

    analytics One row per call leg within a parent call. A single inbound call may have multiple legs (e.g., IVR, transfer to agent). Use for granular call routing and agent-level analysis. Column-level documentation coming soon.

    calls

    analytics One row per inbound or outbound call routed through the facility phone system. Column-level documentation coming soon.

    contact

    analytics One row per contact (tenant, lead, or prospect). A contact may have multiple leases over time. Column-level documentation coming soon.

    cubby_events

    analytics One row per action taken within Cubby by a manager or automated process (e.g., door locked, task changed, competitor tracked, pricing updated). Useful for operational auditing. Column-level documentation coming soon.

    customer_touches

    analytics One row per marketing touchpoint associated with a lead or lease. Links GA session data to contacts and leases to attribute marketing source at the individual customer level. Column-level documentation coming soon.

    discount_programs

    analytics One row per discount program defined by an organization (e.g., Military Discount, Web Special, First Month Free). Column-level documentation coming soon.

    discounts

    analytics One row per discount applied to a lease. Links a lease to the discount program it is enrolled in. Column-level documentation coming soon.

    emails

    analytics One row per email sent through the system (e.g., late notices, move-in confirmations, rate change letters). Column-level documentation coming soon.

    facilities

    analytics One row per facility. The primary dimension table for facility-level analysis. Column-level documentation coming soon.

    facility_groups

    analytics One row per facility-to-group assignment. Used to roll up multiple facilities into named reporting groups. Column-level documentation coming soon.

    ga_events

    analytics One row per Google Analytics 4 event fired on the organization's website. Source data is the GA4 BigQuery export. Column-level documentation coming soon.

    leads

    analytics One row per lead (rental inquiry). A lead is created when a prospective tenant expresses interest in renting a unit. Column-level documentation coming soon.

    lease_auctions

    analytics One row per lease that has entered the lien/auction process. Links a delinquent lease to a specific auction event. Column-level documentation coming soon.

    leases

    analytics One row per lease. The central fact table for tenant and occupancy analysis. Many contact_* fields are denormalized copies of the tenant's current contact record. Column-level documentation coming soon.

    managers

    analytics One row per manager or staff member with access to the Cubby system. Column-level documentation coming soon.

    payments

    analytics One row per payment transaction. Includes payments, refunds, and reversals. Column-level documentation coming soon.

    pricing_group

    analytics One row per pricing group (unit type) per facility. Defines the dimensions, rates, and features for a category of units.
    Column Type Description
    pg_id STRING Unique identifier for the pricing group. Primary key.
    org_id STRING The organization identifier. Common filter field.
    name STRING The name of the pricing group (e.g., "10x10 Drive-Up", "Climate Control Large").
    standard_rate NUMERIC The standard/street rental rate for this unit type.

    rate_changes

    analytics One row per scheduled or applied rent rate change on a lease. Includes full tenant and lease context for operational use. Column-level documentation coming soon.

    scheduled_move_outs

    analytics One row per scheduled move-out notice. Created when a tenant gives notice of their intent to vacate. Column-level documentation coming soon.

    spaces_historical

    analytics One row per unit per day. A daily snapshot of every unit's occupancy, rentability, rate, and tenant status. Use for occupancy trend analysis and historical reporting. Column-level documentation coming soon.

    unit_turnover

    analytics One row per move-in, move-out, or transfer event. Captures the full unit and lease context at the time of the move for cohort and turnover analysis. Column-level documentation coming soon.

    units

    analytics One row per unit. The current state of each storage unit including its dimensions, pricing, and occupancy status. Column-level documentation coming soon.
